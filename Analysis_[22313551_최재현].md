### Use Case #1: System Access (시스템 접속)

**GENERAL CHARACTERISTICS**

| 항목 | 내용 |
| :--- | :--- |
| **Summary** | 교사(관리자)가 티칭오션 매니지먼트의 기능(크롬 확장 프로그램)을 사용하기 위해 시스템에 접속하고 계정을 연동하는 기능 |
| **Scope** | Teaching Ocean Management (Chrome Extension) |
| **Level** | User level |
| **Author** | 최재현 |
| **Last Update** | 2026-05-05 |
| **Status** | Analysis |
| **Primary Actor** | Teacher (교사) / Admin (학원 관리자) |
| **Preconditions** | 1. 크롬 브라우저에 '티칭오션 학생관리 확장 프로그램'이 설치 및 활성화되어 있어야 한다.<br>2. 티칭오션 웹사이트의 유효한 계정을 소유하고 있어야 한다. |
| **Trigger** | 브라우저 우측 상단의 티칭오션 확장 프로그램 아이콘을 클릭했을 때 |
| **Success Post Condition** | 시스템 연동 및 인증에 성공하여 확장 프로그램의 메인 기능(책 종류 갱신, 책 진행상황 조회 등)을 사용할 수 있게 된다. |
| **Failed Post Condition** | 시스템 인증에 실패하여 확장 프로그램의 기능들을 사용할 수 없다. |

<br>

**MAIN SUCCESS SCENARIO**

| Step | Action |
| :---: | :--- |
| **S** | 교사가 크롬 브라우저 상단의 확장 프로그램 아이콘을 클릭하여 팝업창을 열며 시작된다. |
| **1** | 시스템은 현재 크롬 브라우저에 로그인되어 있는 티칭오션 웹사이트의 세션(Session) 데이터를 확인하거나, 확장 프로그램 내 로그인 화면을 띄운다. |
| **2** | (로그인 화면일 경우) 교사는 아이디와 비밀번호를 입력하고 로그인 버튼을 누른다. |
| **3** | 시스템은 해당 계정 정보(또는 세션)를 데이터베이스(Supabase 등)와 대조하여 유효한 관리자/교사 계정인지 확인한다. |
| **4** | 인증이 성공적으로 완료되면 확장 프로그램 메인 화면(기능 버튼 활성화 상태)으로 전환되며 Use case가 끝이 난다. |

<br>

**EXTENSION SCENARIOS**

| Step | Branching Action |
| :---: | :--- |
| **1** | **1a. 티칭오션 웹사이트(teachingocean.co.kr) 세션이 만료된 경우**<br>&nbsp;&nbsp;&nbsp;&nbsp;1a.1. 확장 프로그램에서 로그인이 필요하다는 안내 문구를 띄운다.<br>&nbsp;&nbsp;&nbsp;&nbsp;1a.2. 교사가 로그인 페이지로 이동하여 로그인하도록 유도한다. |
| **2** | **2a. 아이디 또는 비밀번호가 입력되지 않은 경우**<br>&nbsp;&nbsp;&nbsp;&nbsp;2a.1. 아이디와 비밀번호를 모두 입력해 달라는 경고 메시지를 띄운다. |
| **3** | **3a. 등록된 계정이 아니거나 비밀번호가 틀린 경우**<br>&nbsp;&nbsp;&nbsp;&nbsp;3a.1. 일치하는 회원 정보가 없음을 메시지로 띄워준다.<br>**3b. 일반 학생 계정으로 접근하는 경우 (권한 부족)**<br>&nbsp;&nbsp;&nbsp;&nbsp;3b.1. "관리자 및 교사 전용 기능입니다"라는 팝업창을 띄우고 접속을 차단한다. |

<br>

**RELATED INFORMATION**

| 항목 | 내용 |
| :--- | :--- |
| **Performance** | <= 3초 (아이콘 클릭 후 화면이 렌더링되거나 인증이 처리되는 시간) |
| **Frequency** | Variable |
| **Concurrency** | 제한 없음 |

### Use Case #2: Auto-sync Student Data (학생 정보 자동 동기화)

**GENERAL CHARACTERISTICS**

| 항목 | 내용 |
| :--- | :--- |
| **Summary** | 교사가 티칭오션 사이트의 '학생관리' 페이지에 접속하면, 확장 프로그램이 백그라운드에서 학생 목록을 스크래핑하여 데이터베이스(Supabase)를 최신 상태로 동기화하는 기능 |
| **Scope** | Teaching Ocean Management (Chrome Extension) |
| **Level** | User level (Background process) |
| **Author** | 최재현 |
| **Last Update** | 2026-05-05 |
| **Status** | Analysis |
| **Primary Actor** | Teacher (교사) / Admin (학원 관리자) |
| **Preconditions** | 1. 확장 프로그램이 활성화되어 있고 로그인이 완료된 상태여야 한다.<br>2. 티칭오션 웹사이트에 관리자 계정으로 접속 중이어야 한다. |
| **Trigger** | 티칭오션 웹사이트 내의 '학사관리 ➔ 학생관리(또는 학생 목록)' 페이지로 이동하여 화면이 로드되었을 때 |
| **Success Post Condition** | 화면의 학생 데이터가 성공적으로 추출되어 DB(Supabase)에 최신 상태로 동기화된다. |
| **Failed Post Condition** | 네트워크 오류나 웹사이트 구조 변경 등으로 인해 스크래핑 및 DB 동기화에 실패한다. |

<br>

**MAIN SUCCESS SCENARIO**

| Step | Action |
| :---: | :--- |
| **S** | 교사가 티칭오션 웹사이트의 '학생관리' 페이지에 접속하며 시작된다. |
| **1** | 확장 프로그램(Content Script)이 해당 페이지의 DOM(문서 구조) 로드 완료를 감지한다. |
| **2** | 화면에 표시된 학생 리스트 데이터(학생 이름, 소속, 현재 레벨 등)를 백그라운드에서 스크래핑하여 JSON 형태로 변환한다. |
| **3** | 스크래핑한 데이터를 서버(DB)로 전송하여, 기존에 저장된 데이터베이스 정보와 대조한다. |
| **4** | 신규 등록된 학생이나 레벨이 변경된 학생 등 변경점(Diff)을 찾아내어 데이터베이스에 일괄 갱신(Bulk Update/Insert)한다. |
| **5** | 백그라운드 동기화 작업이 완료되며 Use case가 끝이 난다. |

<br>

**EXTENSION SCENARIOS**

| Step | Branching Action |
| :---: | :--- |
| **1** | **1a. 웹페이지 로딩이 비정상적으로 지연되거나 무한 로딩이 걸린 경우**<br>&nbsp;&nbsp;&nbsp;&nbsp;1a.1. 설정된 대기 시간(Timeout)을 초과하면 동기화 시도를 안전하게 취소한다.<br>&nbsp;&nbsp;&nbsp;&nbsp;1a.2. 다음 번 해당 페이지 접속 시 다시 동기화를 시도한다. |
| **2** | **2a. 티칭오션 원본 웹사이트의 UI(HTML 태그 구조 등)가 패치되어 변경된 경우**<br>&nbsp;&nbsp;&nbsp;&nbsp;2a.1. 타겟 요소를 찾지 못해 스크래핑에 실패한다.<br>&nbsp;&nbsp;&nbsp;&nbsp;2a.2. 에러 로그를 시스템에 남기고 동기화 프로세스를 중단한다. |
| **4** | **4a. 데이터베이스(Supabase) 접속 지연이나 네트워크 통신 에러가 발생한 경우**<br>&nbsp;&nbsp;&nbsp;&nbsp;4a.1. 데이터 갱신을 취소하고 원래 상태로 롤백(Rollback) 처리한다. |

<br>

**RELATED INFORMATION**

| 항목 | 내용 |
| :--- | :--- |
| **Performance** | <= 5초 (페이지 로드 후 백그라운드 데이터 파싱 및 DB 저장 완료까지) |
| **Frequency** | Variable (해당 페이지에 들어갈 때마다 백그라운드에서 자동 실행됨) |
| **Concurrency** | 제한 없음 |
| **Due Date** | |
